Version 4.4.6

2022-08-02 - Happy New Month!
Added an in-game chat
Here you can switch between two modes: Lobby and Squad
In the Lobby mode, you can chat with everyone in the same match as you
In the Squad mode, you can only chat with people who have the same squad code as you
You can receive kill feeds, victory feeds and player join/disconnect feedback!
Note that chat is restricted to accounts only, and that there is a swear filter, so please be careful.

Version 4.4.0

2022-05-21 - Content Patch
Spectator mode
Finally added spectator mode
Once you get a game over spectator mode is automatically enabled
As the name says, you can spectate anyone in game
Once everyone else is dead or you're the last person to get a game over the regular game over sequence occurs
Supports mobile controls too!
Editor update
Thanks to Nightcat (again), the editor now has an UI inspired by Windows 11
The editor is currently restricted to light theme, however a future update will add a settings button where you can change the theme and certain other things, such as whether to use middle click or left and right click, etc...
Please look forward to it.
Improved custom assets
In the editor you can now set an url to use for assets instead of being limited to the github repository
You can still use the original assets system by just typing in the json name
Fixed a bug in which assets would get cleared when saving your world
Mobile changes
The hud buttons in the top right can finally be used!
The joystick can now only be activated from the left hand side of the screen
You can no longer jump to return to the lobby
It didn't really work, it only supported regular jumps and didn't work on mobile, which is why I'm removing it.
Updated control rebinding app
Added a new "Clear" button next to every control. This button clears the binding of that specific control.
When setting a control, the color of the control changes, like the headers.
Added cooldown to multi coin blocks
In the original SMB1, after you hit a multi coin block, the block had a ~270ms cooldown, this has been added to Mario Royale to remain faithful to the original NES classic.
General improvements and fixes
You can no longer update your nickname if it's less than 4 letters long, or if it's more than 20 letters long.

Version 1.0.1

2019-06-16 - Performance Patch
Player count reduced to 75 from 99
Both the server and the game client struggle to handle a full 99 players so I'm reducing it for now to help alleviate performance issues on both the backend server and the client renderer.
I also feel like 99 players is a bit overly chaotic right at the start, so hopefully this will make it more manageable.
After all... Most battle royales spread their players over a large area. This game dumps them all in 1 place.
Renderer culling changes
Small change to slightly improve framerates.
Experimental Server Performance Improvements
Made some changes to how the server writes game data to clients. Should be thread safe but may cause problems. I'll roll this back if it breaks things.
Improved support for browsers that don't have a gamepad api
Hopefully fixes a few web browsers that like... no one uses...
Fixed num0 key being wrong keycode

Added more stuff you can't get in the lobby

Planned stuff:
Hopefully will be able to add some new features soon.
Most requested features are: touch controls, squads, and rebindable controls.
Please look forward to it.
